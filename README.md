# Memory Card

Веб-приложение для эффективного запоминания терминов и слов с помощью карточек. Пользователи создают карточки, объединяют их в модули и группы, проходят тренировки и отслеживают прогресс.

---

## Основная идея

- Карточки живут в общем словаре и могут быть объединены в модули и группы.  
- Доступны 4 режима тренировок:  
  1. Повторение  
  2. Заучивание (вариант из 4)  
  3. Тест на ввод  
  4. Соответствия  
- В тренировках можно включить просмотр предложения с термином.  
- Есть встроенный переводчик для карточек.  
- Пользователи могут публиковать модули и группы в сообществе.  
- Хранится статистика для визуализации прогресса.

Цель MVP: быстрый, минималистичный, mobile-friendly интерфейс и стабильный API на стеке **PERN** (Postgres + Express + Node + React).

---

## Технологии

- **Frontend:** React + TypeScript  
- **Backend:** Node.js + Express  
- **База данных:** PostgreSQL  
- **Аутентификация:** JWT + bcrypt  
- **Хранение иконок:** frontend `public/icons/`
- **ORM/миграции:** Sequelize  

---

## Страницы и экраны

1. **Главная:** список групп пользователя, прогресс, блок «Мои модули».  
2. **Community:** публичные модули/группы, рейтинг, кнопка «Добавить к себе».  
3. **Dictionary:** список всех карточек, поиск, фильтр по тегам, переводчик.  
4. **Group page:** список модулей группы, вкладки Modules / Statistics.  
5. **Module page:** список карточек модуля, редактирование, вкладки Cards / Statistics.  
6. **Module/Group Statistics:** графики прогресса, линейные, барчарт, круговые, тепловые карты.  
7. **Publish flow:** модалка публикации с модерацией.

---

## Юзерфлоу

1. Создание карточки → добавление тегов, примеров, перевода.  
2. Объединение карточек в модули → объединение модулей в группы.  
3. Запуск тренировки → выбор модуля(ей) → прохождение режима.  
4. Сохранение результатов → обновление статистики → кнопка «Повторить забытые».  
5. Просмотр статистики по модулю, группе или общая.  
6. Публикация модуля/группы → Community.

---

## Структура базы данных (основные таблицы)

- **User, Card, Tag, Module, Group** — сущности пользователей, карточек, тегов, модулей и групп.  
- **ModuleCard, GroupModule** — связи «модуль → карточка», «группа → модуль».  
- **Training, TrainingModule, DailyCardStats** — для тренировок и статистики.  
- **Publication** — публикации в сообществе.  

*(Полная структура и поля в исходном коде репозитория)*

## Особенности UX

- Создание карточки в модуле = сохранение в словарь + добавление в модуль.  
- Перевод через встроенный API, возможность редактирования перед добавлением.  
- Статус «выученности» = уровень + история/агрегаты.  
- Повторение забытых карточек после тренировки.

---

##  Запуск проекта локально

1. `npm install` в папках frontend и backend  
2. Настроить .env с параметрами БД и JWT  
3. `npm run dev` в папке backend
4. `npm start` в папке frontend
5. Открыть [http://localhost:3000](http://localhost:3000)
